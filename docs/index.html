Agentwright

for Claude Code

Not tokens burned — harness maturity.

Agentwright measures how well you work with Claude Code, not how much you spend on it — and coaches you by turning your real, recurring friction into rules, hooks, and gates, then verifying that each fix actually worked. Everything runs locally.

What it measures

Mechanics the model performs for you aren't graded. Four things that grow more valuable as models improve are:

Landscape

Do you know a lever exists — hooks, gates, MCP, subagents, plan mode?

Judgment

Given a real friction, do you pick the appropriate lever — and justify it by consequences?

Oversight

Do you catch quiet defects where errors are expensive — security, harness, irreversible actions?

Outcome

Is there an appropriate, working, safe setup — regardless of who built it?

A deliberate "no" earns full credit

Don't use a lever, but can explain the tradeoff that rules it out in your context? Full credit, marked as a conscious refusal. Seniority is closed consciously, not everything installed. Try it yourself in the score simulator below.

Anti-gaming rules

Fresh artifactsAn empty skill created minutes before scoring counts partial at best — usage evidence is needed for full credit.
Refusal quality bar"I don't need it" without a why = absent. The refusal must show you know what the lever guarantees and what it costs.
Score can fallA lever that didn't work, growing friction, a regressed permission — the number goes down, with the reason stated.
No mercy rounding84.4 is 84. Level boundaries are never rounded across.

Commands

/agentwright:score

Deterministic scan → checklist → short situational dialog → weighted score with security gate → persistent scorecard. Ends with up to three growth items — only real gaps, never padding — each tied to a practical task in your own repo (and a free Anthropic Academy module when a fitting one exists).

/agentwright:coach

The closed loop you clicked through above — verify past fixes (rates per 100 turns of activity), group frictions, nominate opportunities, your judgment first, apply safely from a vetted lever library, record. Also triggers when you complain about recurring friction.

/agentwright:log

One-line friction capture in the moment: "this took three tries again". No hook can see what annoyed you — manual notes are the highest-signal input, quoted verbatim when the coach groups frictions.

/agentwright:show

Instant read-only display of the saved score — screen-share friendly: headline, axes, per-project lines, trend. Every score run seals the card; a scorecard edited by hand is refused, not displayed. Self-assessed, and the card says so.

Privacy boundary

Everything is local; nothing is transmitted anywhere. The boundary is architectural — the forbidden side isn't just unused, the code contains no path that reads it (per the Anthropic Software Directory Policy, §1.D/§1.F).

✓ Reads

  • ✓Its own journal — event kind, tool name, failure category, project name, timestamp, plus notes you log yourself
  • ✓Your config artifacts — settings.json, CLAUDE.md, hooks, skills, agents, .mcp.json
  • ✓Repo state — .gitignore coverage, CI (GitHub or GitLab), pre-commit config, secrets scan; a multi-repo workspace root is scanned per repo, personal non-git folders are left alone
  • ✓Its own scorecard — score history and recorded actions

✕ Never touches

  • ✕Conversation transcripts — ~/.claude/projects/
  • ✕Usage analytics data — ~/.claude/usage-data/, insights reports, session summaries
  • ✕Prompts, tool inputs/outputs, error text — not even for logging
  • ✕The network — no telemetry, no uploads; delete ~/.claude/agentwright/ to erase everything

Install & quickstart

Requirements: Claude Code, bash, python3. Optional: git and gitleaks sharpen the probes.

# development / local use — active only in sessions started this way
claude --plugin-dir /path/to/agentwright

# or install from the marketplace
/plugin marketplace add Victor-Borisov/agentwright
/plugin install agentwright@agentwright

Then run /agentwright:score for your baseline. Hooks start journaling from that moment; give the coach a day or two of real work before the first /agentwright:coach.

FAQ

Will it replace or break my existing statusline?

No. Plugins cannot install statuslines in Claude Code — a statusline is configured only in your own settings.json, which Agentwright never modifies. scripts/statusline-example.sh is an optional example: if you want the score in your existing statusline, append a few lines to your current script instead of replacing it.

Does it send my score or data anywhere?

No. Everything is local. The plugin makes no network calls and uploads nothing, ever. You can verify this: the entire codebase is a few short scripts.

I don't use worktrees / MCP / plan mode. Will my score suffer?

Not if it's a choice. Explain the tradeoff that rules the lever out in your context and the item is credited in full as a conscious refusal — try it in the simulator above. The score punishes only the levers that aren't on your map at all — and then gives you a targeted way to learn each one.

Why did my score go down?

Three possible reasons: a security regression (gate cap), a past fix that verifiably didn't work (Step 0), or friction growing while the setup stagnates. The report always names the cause. A falling number you can explain beats a flattering one you can't.

Why doesn't it analyze my conversations to coach me better?

Two reasons. Policy: directory-listed software must not read Claude's chat history or conversation summaries. Method: behavior visible in artifacts and outcomes (did friction fall?) is verifiable; judging your prose is guesswork. Agentwright grades what can be proven.

Where is my data, and how do I reset?

All of it lives in ~/.claude/agentwright/ — journal, flags, scorecard, archive, config (your dialog language, asked once at the first score or coach run). Delete that directory and Agentwright starts from zero.
